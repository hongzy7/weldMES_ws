# 机器人数据接口读取说明

本文说明“华沿机器人智能焊接助手”后续与机器人控制器、焊接工艺包打通时，优先读取哪些状态参数，以及建议从哪个协议入口读取。

## 推荐接口分层

智能焊接助手不要直接在前端访问机器人接口，建议后续增加一个后端适配层：

```text
智能焊接助手
  -> Robot Adapter Service
    -> 机器人控制器接口
    -> 焊接工艺包接口
    -> 焊机/焊接设备实时数据接口
```

第一阶段建议只做读取和监控，不直接开放运动控制、起弧、暂停、恢复等写操作。

## 控制器状态读取

控制器通信建议优先使用 Python SDK；如果需要更底层控制，再直接使用 TCP IF 指令。

### 连接方式

- 控制器 IF 指令服务端口：`10003`
- 控制器协议格式：`指令名,参数1,参数2,;`
- Python SDK 连接接口：`HRIF_Connect(boxID, IP, nPort)`
- 常用默认端口：`10003`

### 机器人状态

用于工作台设备状态、自检状态、报警状态、任务是否完成。

控制器 IF 指令：

```text
ReadRobotState,nRbtID,;
```

成功返回包含：

- 运动状态 `nMovingState`
- 使能状态 `nEnableState`
- 错误状态 `nErrorState`
- 错误码 `nErrorCode`
- 错误轴 `nErrorAxis`
- 抱闸状态 `nBreaking`
- 暂停状态 `nPause`
- 急停状态 `nEmergencyStop`
- 安全光幕状态 `nSaftyGuard`
- 上电状态 `nElectrify`
- 电箱连接状态 `nIsConnectToBox`
- 路点运动完成状态 `nBlendingDone`
- 到位状态 `nInPos`

建议写入系统：

- `焊前准备数据.机器人状态`
- `焊接过程数据.报警码`
- `BI指标输出.报警次数`

## TCP 位置与速度读取

### TCP 实际位置

用于记录机器人焊接过程中的实际位置、路径轨迹、作业状态。

控制器 IF 指令：

```text
ReadActPos,nRbtID,;
```

成功返回包含：

- 关节实际位置：`dJ1` ~ `dJ6`
- 空间实际位置：`dX`, `dY`, `dZ`, `dRx`, `dRy`, `dRz`
- 当前工具坐标：`dTcp_X`, `dTcp_Y`, `dTcp_Z`, `dTcp_Rx`, `dTcp_Ry`, `dTcp_Rz`
- 当前用户坐标：`dUcs_X`, `dUcs_Y`, `dUcs_Z`, `dUcs_Rx`, `dUcs_Ry`, `dUcs_Rz`

建议写入系统：

- 原始过程采样表
- `焊接过程数据.实际焊接长度`
- 焊缝追溯详情中的轨迹记录

### TCP 实际速度

用于显示 TCP 当前速度、判断机器人是否稳定运动、计算焊接过程速度。

控制器 IF 指令：

```text
ReadActTcpVel,nRbtID,;
```

成功返回：

- `dX`, `dY`, `dZ`：TCP 实际线速度，单位 `mm/s`
- `dRx`, `dRy`, `dRz`：TCP 实际角速度，单位 `°/s`

如果要显示一个综合 TCP 线速度，可由适配层计算：

```text
tcp_speed = sqrt(dX^2 + dY^2 + dZ^2)
```

### TCP 命令速度

用于对比设定/命令速度与实际速度。

控制器 IF 指令：

```text
ReadCmdTcpVel,nRbtID,;
```

返回字段同 `ReadActTcpVel`，但含义是命令速度。

### 速度倍率

用于显示当前系统速度倍率。

控制器 IF 指令：

```text
ReadOverride,nRbtID,;
```

返回：

- `dOverride`：当前速度比，范围 `0.01~1`

建议写入系统：

- `焊接过程数据.实际设定速度`
- BI 工艺符合率计算时的辅助字段

## 路径、走了多远、进度读取

如果后续机器人采用连续轨迹或路径方式执行焊接，建议读取轨迹状态和轨迹进度。

### 当前路点 ID

用于知道机器人当前执行到哪个路点或动作。

控制器 IF 指令：

```text
ReadCurWayPointID,nRbtID,;
```

返回：

- `strCurWaypointID`：当前运动路点 ID

### 轨迹状态

用于判断轨迹是否已计算完成、是否可以运行、是否计算错误。

控制器 IF 指令：

```text
ReadPathState,nRbtID,sPathName,;
```

返回：

- `nStateJ`：MovePathJ 状态
- `nErrorCodeJ`：MovePathJ 错误码
- `nStateL`：MovePathL 状态
- `nErrorCodeL`：MovePathL 错误码

重点状态：

- `0`：轨迹尚未示教
- `1`：轨迹正在示教中
- `2`：轨迹正在目标点位计算中
- `3`：轨迹目标点位计算完成
- `5`：轨迹目标点位计算错误
- `9`：轨迹已导入
- `10`：轨迹导入后处理中

### 轨迹运动进度

用于回答“走了多远”“当前路径进度是多少”。

控制器 IF 指令：

```text
ReadSoftMotionProgress,nRbtID,;
```

返回：

- `dProcess`：当前轨迹运动进度，范围 `0~1`，大于 `0.999999` 可认为完成
- `nIndex`：当前运动到的点位索引

注意：

- 该接口仅对 `MovePathL` 有效，对旧的 `MovePath` 无效。
- 协议建议读取频率保持在 `20ms` 左右。
- 系统中可以保存 `dProcess`，并结合理论焊缝长度计算估算已焊长度：

```text
估算已焊长度 = 理论焊缝长度 * dProcess
```

如果需要更精确的实际路径长度，应由适配层周期读取 `ReadActPos`，对相邻 TCP 位置点做距离累加。

## 焊接工艺包状态读取

焊接工艺包主要用于读取焊接任务状态、焊机设备状态、焊机错误码和实时送丝速度。

### 连接方式

- 焊接控制指令端口：`30600`
- 焊接状态消息队列端口：`30601`
- 状态推送方式：ZeroMQ 发布/订阅
- 状态推送周期：约 `100ms`
- 焊接控制协议格式：`####` + 8 位十六进制长度 + JSON 消息体 + `$$$$`

### 状态推送

订阅 `30601` 后读取 `StatusMessage`。

重点字段：

- `taskID`：正在执行的任务 ID
- `actionID`：正在执行的动作 ID
- `actionResult`：动作结果
- `taskStatus`：任务状态
- `taskType`：任务类型
- `reservationStatus`：预约状态
- `deviceStatus`：设备状态列表
- `weldErrorCode`：焊机错误状态码，`0` 表示焊机正常
- `weldMode`：焊接模式
- `teachMode`：示教模式
- `realtimeWireSpeed`：实时送丝速度
- `laserWeldState`：激光焊接状态，针对特定设备

建议写入系统：

- `焊前准备数据.焊机状态`
- `焊接过程数据.报警码`
- `焊接过程数据.机器人程序号` 或动作 ID 映射字段
- `BI指标输出.报警次数`

## 电流、电压如何稳定读取

### 需要区分设定值和实际值

焊接工艺包中的 `changeArcSet` 是“设置焊接参数”接口，其中包含：

- `arcSetCurrent`
- `arcSetVoltage`
- `arcSetWireSpeed`
- `arcOnCurrent`
- `arcOnVoltage`
- `arcOffCurrent`
- `arcOffVoltage`

这些字段适合作为“推荐/下发/实际设定参数”记录到 MES，但不能直接当作焊机实时采样电流、电压。

建议写入：

- `工艺参数数据.推荐电流`
- `工艺参数数据.推荐电压`
- `焊接过程数据.实际设定电流`
- `焊接过程数据.实际设定电压`
- `焊接过程数据.实际设定速度`

### 实际电流、电压的推荐读取策略

当前焊接工艺包文档明确的 `30601 StatusMessage` 标准字段中，可以稳定读取到焊机状态、焊机错误码和实时送丝速度；文档片段未明确列出实时电流、实时电压字段。

因此建议分两层处理：

1. 标准稳定层：
   - 从 `30601` 读取任务状态、设备状态、焊机错误码、实时送丝速度。
   - 从 `changeArcSet` 或任务模板记录设定电流、设定电压、设定送丝速度。

2. 实际采样层：
   - 如果现场焊机品牌协议通过工艺包扩展字段提供实际电流/电压，则由适配层读取扩展字段并归一化。
   - 如果工艺包没有输出实际电流/电压，则通过焊机自身协议读取，例如 Modbus TCP、EtherNet/IP、RS485 或焊机厂商 SDK。
   - 如果焊机实际电流/电压接到机器人电箱模拟量输入，可用控制器模拟量接口读取，例如 `ReadBoxAI` 或 Python SDK 的对应接口，再按量程换算成电流/电压。

不要把 `arcSetCurrent` / `arcSetVoltage` 当作实时采样值；它们更适合作为设定值。

## 智能焊接助手建议保存的数据

### 机器人状态采样

建议采样频率：`100ms~500ms`，根据工控机性能和业务需求调整。

字段建议：

- 采样时间
- 任务号
- 机器人编号
- 运动状态
- 使能状态
- 错误状态
- 错误码
- 急停状态
- 安全光幕状态
- 到位状态
- TCP 实际位置
- TCP 实际速度
- 速度倍率
- 当前路点 ID
- 轨迹进度
- 轨迹点索引

### 焊接状态采样

建议优先订阅 `30601`，按消息到达写入缓存或过程表。

字段建议：

- 采样时间
- 焊接任务 ID
- 焊接动作 ID
- 任务状态
- 动作结果
- 焊机设备名称
- 焊机型号
- 焊机协议
- 焊机状态
- 焊机错误码
- 焊接模式
- 示教模式
- 实时送丝速度
- 设定电流
- 设定电压
- 实际电流
- 实际电压

其中 `实际电流` 和 `实际电压` 需要根据现场焊机接入方式确认来源，不能默认由 `30601` 标准状态字段提供。

## 建议落地顺序

1. 先接控制器只读接口：
   - `ReadRobotState`
   - `ReadActPos`
   - `ReadActTcpVel`
   - `ReadOverride`
   - `ReadCurWayPointID`

2. 再接轨迹进度：
   - `ReadPathState`
   - `ReadSoftMotionProgress`

3. 再接焊接工艺包状态：
   - 订阅 `30601 StatusMessage`
   - 读取 `deviceStatus`
   - 读取 `weldErrorCode`
   - 读取 `realtimeWireSpeed`

4. 最后接焊机实际电流/电压：
   - 优先确认工艺包是否提供对应扩展字段。
   - 如无扩展字段，从焊机协议或模拟量输入读取。
